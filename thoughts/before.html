Before we begin

I'm not a developer.

I do have a strong foundation in development, however. I grew up tinkering with SCSI cables and MS-DOS scripts with my Dad. I learned BASIC in high school, and have built way too many static sites over the years. Heck, I even took a course in Front End Development at Treehouse a few years back.

But it's like the saying goes: "If you don't use it, you lose it." Looking back, I found myself losing interest when trying to master floats, or giving up the ghost when I couldn't find a way through. Or maybe a freelance project ended and my need to use Git on a daily basis waned heavily.

My hope is that this site will become a sandbox for me to explore code, and push my strengths as a writer too. So to start things off, I thought I'd explain what tech I'm using and how to get started with it.

A guide to Jekyll

What you'll learn

  • Basic HTML, CSS and SASS
  • Basic command line
  • How to use Git, and Github
  • What Jekyll is, and how to install it

So, what is Jekyll?

Basically Jekyll is a "static site generator" that runs on a programming language called Ruby. For the most part, working with Jekyll is made up of Markdown, HTML, CSS and SASS files. In fact, you won't mess with Ruby at all. Jekyll is meant for blogging, and comes pre-installed with folders for "posts" and "drafts", but you can use it for anything.

Did you say, static?

Yeah. So websites fit into two camps: dynamic and static. A dynamic site has a back-end database attached to it, most often written in PHP. Dynamic sites, like Wordpress, act like a CMS (Content Management System) so maintaining the site is ultra-simple. Static, on the other hand, is basically a collection of folders and files, with a few bonus features.

Oooo. What bonus features?

Jekyll is a powerfully-free platform to use once you get over the small learning curve. Some of these bonus features include: converting Markdown files to HTML, a built-in process for converting CSS to SASS, and something else called apartial. Basically, a partial is a file that includes dense information (like the contents of the head) and stores this file into a folder called an includes. Overall, the power of Jekyll is that it's free, fast, simple and flexible. But it isn't intended for client work because the upkeep is made for a developer.

What about Github?

Github pairs well with Jekyll because you can host your site, or repo, on it for free, and creating a repo to store your site allows you to use git to track changes. There is something called a Github page which converts your repo into a website (like mysite.github.io). Learn more about that last bit here.

How do I get started?

There are a loads of sites dedicated to getting started, but I found Tania's site to be the most approachable. She does a better job than I can about what you need to get started, whether that's installing Xcode Tools or making sure you have bundle or the right gems installed. At it's most basic, you'll want to: 

  // Create a repo or Github page
  cd /Users/jon/Sites/

  // Clone repo to your machine
  git clone git@github.com:jonaschafer/kiddos.git

  // Install Jekyll into that repo folder
  jekyll new kiddos --force`

  // Launch Jekyll's server
  jekyll serve --livereload
