Before we begin

I'm not a developer.

I do have a strong foundation in development, however. I grew up tinkering with SCSI cables and MS-DOS scripts with my Dad. I learned BASIC in high school, and have built way too many static sites over the years. Heck, I even took a course in Front End Development at Treehouse a few years back.

But it's like the saying goes: "If you don't use it, you lose it." Looking back, I'd stop messing around with code when I got stuck (and couldn't see a way through). Or a freelance project would end and needing to use Git on a daily basis waned heavily.

This site is meant to be a sandbox to help me keep at learning code. That could mean changing the design of the site, trying out a CSS experiment, or just writing more. So, before we begin, I wanted to kick off this post with a hopefully-quick intro to how I set up this site on Jekyll.

Goals of this post

  • Keep up with basic HTML, CSS and SASS

So, what is Jekyll?

Basically Jekyll is a "static site generator" that runs on a programming language called Ruby. For the most part, working with Jekyll is comprised of Markdown, HTML, CSS and SASS files. In fact, you won't mess with Ruby at all. Jekyll is meant for blogging, and comes pre-installed with folders for "posts" and "drafts", but you can use it for anything.

Did you say, static?

Nice catch. Websites fit into two camps: dynamic and static. A dynamic site has a back-end database attached to it, most often written in PHP. Dynamic sites, like Wordpress, act like a CMS (Content Management System), so maintaining the site is ultra-simple. Static, on the other hand, is basically a collection of folders and files, with a few bonus features.

Oooo. What bonus features?

Jekyll has a bunch of little tricks that allows you to use it like a lo-fi CMS. For instance, it converts Markdown files to HTML, CSS to SASS (it has a built-in processor), and allows you to use something called a partial. A partial is a file that includes dense information (like the contents of the head) and stores this file into a folder called an includes.
